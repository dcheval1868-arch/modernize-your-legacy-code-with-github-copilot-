## 補足

- `Actual Result`、`Status (Pass/Fail)`、`Comments` はテスト実施時に記入してください。
- 将来の Node.js テスト化では、次の観点で分割すると移行しやすくなります。
  - 単体テスト: 入金/出金/残高チェックの純粋ロジック
  - 結合テスト: メニュー入力から残高更新までの一連フロー
  - 回帰テスト: 残高不足時に残高が不変であること
